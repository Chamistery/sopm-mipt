# ADR 0002 — Авторизация в dev-режиме через X-User-Id / X-User-Role

**Дата:** 2026-05-03
**Статус:** Принято

## Контекст

OAuth МФТИ ещё не подключён. Бэк (`backend/project-service/internal/auth/auth.go`,
`internal/middleware/auth.go`) определяет текущего пользователя из заголовков
`X-User-Id` (число) и `X-User-Role` (одна из `student`, `teamlead`, `mentor`,
`coordinator`, `admin`). Если заголовков нет или ID невалидный — пользователь
анонимный.

Это полностью **dev-режим** — заголовки тривиально подделать. Прод этого
сервиса работать не должен, пока не будет настоящего OAuth.

## Решение

1. На фронте есть страница `/login` со списком всех пользователей из
   `GET /api/users`. Пользователь кликает «Войти как X», мы сохраняем
   `{userId, role, displayName}` в `localStorage` под ключом `sopm.auth.v1`.
2. `apiFetch` (и сгенерированный SDK через interceptors) автоматически
   подставляет эти заголовки в каждый запрос. Источник — функция
   `setAuthHeaderProvider`, которую регистрирует `auth/store.ts`.
3. В сайдбаре есть `RoleSwitcher` — позволяет на лету сменить роль для текущего
   юзера (тестовый режим, для демо). В продакшене этот компонент должен быть
   скрыт по env-флагу.
4. После реального OAuth МФТИ:
   - `LoginPage` редиректит на OIDC-провайдер вместо рендера списка.
   - Колбэк OIDC сохраняет `{userId, role}` в тот же store.
   - `apiClient` остаётся неизменным — он не знает про OAuth, только про
     результат: `userId` и `role`. Заголовки `X-User-Id`/`X-User-Role` уйдут,
     заменятся `Authorization: Bearer ...`. Хелпер `setAuthHeaderProvider`
     возвращает Map, расширяемая под bearer.
   - `RoleSwitcher` скрывается флагом `import.meta.env.PROD`.

## Последствия

- Демо/разработка возможна без OAuth.
- Пользователь может представиться **любым** юзером — это намеренно для dev'а.
- Любой агент, реализующий фичу, может полагаться на `useCurrentUser()` и не
  думать про авторизацию: либо она есть и хук возвращает юзера, либо
  `RequireAuth` перебрасывает на `/login`.

## Альтернативы

- _Не делать `LoginPage`, держать выбор юзера в URL/cookie вручную._
  Тяжелее тестировать UX.
- _Сразу подключать OAuth._ OAuth МФТИ — отдельная задача, ждёт
  административного согласования.
