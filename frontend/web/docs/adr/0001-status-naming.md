# ADR 0001 — Cтатусы и идентификаторы по контракту бэка

**Дата:** 2026-05-03
**Статус:** Принято

## Контекст

В прототипах (`frontend/prototypes/common.js`) задачи и заявки имели «фронтовые»
строковые статусы: `pending_approval`, `approved`, `progress`, `review`,
`returned`, `done`, `rejected`. Бэк (`backend/project-service`) использует
кириллицу: `Ожидает аппрува`, `Назначена`, `В работе`, `На ревью`, `Возвращена`,
`Готово`, `Отклонена` (см. `database_changes.md`).

Раньше планировалось писать словарь-маппер. Теперь, когда фронт и бэк делает
одна команда, мы можем зафиксировать единый словарь и не писать трансляторы.

## Решение

1. **Статусы передаются и хранятся как в БД, по-русски**. Фронт получает и
   показывает их без перевода (оба слоя — русскоязычные).
2. **Идентификаторы — числовые `id` из БД**, везде. Никаких строк-ников
   («Кузнецов М.») в полях бизнес-данных. Display name — производное,
   формируется хелпером `fullName(user)` или приходит готовым в API-ответах.
3. **Даты — ISO 8601 строки** (`YYYY-MM-DD` для дат, RFC 3339 для timestamp'ов),
   как их отдаёт Go. Хранение в стейте — те же строки. Парсим в `Date` только
   на месте использования (Гант, форматтер).
4. **`start`/`dur` (день от начала спринта + длительность) — производные.**
   На входе/выходе у API только `startDate`/`endDate`. Расчёт `start`/`dur`
   происходит локально в Гант-компоненте, в стейт не утекает.

## Последствия

- На фронте нет словаря «backend → frontend» статусов. Меньше кода — меньше
  багов синхронизации.
- Если бэк добавит новый статус — TypeScript ругаться не будет (статусы
  типизированы как union из строк, обновляется кодгеном из swagger.yaml).
- Задачи в Storybook принимают `startDate`/`endDate`, не `start`/`dur`.

## Альтернативы

- _Оставить латиницу на фронте + словарь._ Больше кода, синхронизация двух
  словарей, риск рассинхрона при добавлении нового статуса.
- _Перевести бэк на латиницу._ Большая миграция БД (`UPDATE tasks SET status = ...`),
  сломает существующие сидовые/демо-данные, не даёт ничего полезного.
